## Google検索だけでは足りなくなっている

ChatGPT、Perplexity、Gemini、Copilot — ユーザーの「調べる」行動が変わりつつあります。

従来のSEOは「Googleの検索結果で上位に表示されること」がゴールでした。しかし2025年以降、ユーザーの情報取得チャネルは確実に分散しています。

- **Google検索**: 依然として最大の流入元。ただしAI Overviewの導入で検索結果ページの構造が変化
- **AI検索（ChatGPT、Perplexity等）**: 「〇〇 おすすめ」「〇〇 比較」系の検索でAIの回答を参照するユーザーが増加
- **Google AI Overview**: 検索結果の上部にAIが生成した回答が表示。ここに引用されるかどうかが新たな競争軸

つまり、**同じFAQコンテンツでも、届け先が3つに分かれている**のが現状です。

1つの流入経路に最適化するだけでは、残りの2つを取りこぼします。

## AEOとLLMO — SEOの拡張概念

まず用語を整理します。

| 用語 | 正式名                            | 最適化対象                            |
| ---- | --------------------------------- | ------------------------------------- |
| SEO  | Search Engine Optimization        | Google・Bing等の検索エンジン          |
| AEO  | Answer Engine Optimization        | Google AI Overview・Featured Snippet  |
| LLMO | Large Language Model Optimization | ChatGPT・Perplexity・Gemini等のAI検索 |

**AEO**は、Googleの検索結果に「回答」として表示されることを狙う最適化です。Featured Snippet（強調スニペット）やAI Overviewに引用されるには、質問と回答の構造が明確なコンテンツが有利です。

**LLMO**は、LLM（大規模言語モデル）があなたのサイトを正しく理解し、回答に引用してくれるよう最適化することです。LLMはHTMLをそのまま読むより、構造化されたテキスト（Markdown、JSON-LD等）のほうが効率的に情報を取得できます。

この3つは別々の施策ではなく、**同じコンテンツを3つのチャネルに届ける設計**で同時に対応できます。

## 3チャネル戦略の全体像

以下の3面にFAQコンテンツを展開します。

```
同じFAQコンテンツ
    ├── Google向け:      JSON-LD FAQPage構造化データ
    ├── ユーザー+Google向け: ページ表示FAQ（HTML）
    └── AI検索向け:      llms.txt / llms-full.txt
```

それぞれの役割を詳しく見ていきます。

### チャネル1: JSON-LD FAQPage構造化データ（Google向け）

JSON-LD形式のFAQPage構造化データは、Googleが**機械的に質問と回答の構造を認識する**ための仕組みです。

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "他のSEOツールとの違いは？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "高額ツールは被リンク分析・順位追跡に特化しています。当ツールは診断→改善コード自動生成に特化しています..."
      }
    }
  ]
}
```

**効果:**

- Google検索結果でFAQリッチリザルトとして展開表示される可能性
- AI Overviewの情報ソースとして参照される
- 「〇〇 vs △△」「〇〇 代わり」といった比較検索で引用されやすい

### チャネル2: ページ表示FAQ（ユーザー+Google向け）

JSON-LDはHTMLの`<head>`に埋め込むため、ユーザーの目には見えません。**同じFAQ内容をページ上にも表示する**ことで、2つ目の効果を得ます。

```tsx
// アコーディオン形式でFAQを表示
const faqs = [
  {
    question: "他のSEOツールとの違いは？",
    answer: "高額ツールは被リンク分析・順位追跡に特化しています...",
  },
  // ...
];
```

**効果:**

- ユーザーの疑問にページ内で直接回答 → 直帰率の低下
- Googleのテキスト解析でもFAQ内容が認識される（JSON-LDとの二重効果）
- 「〇〇 比較」「〇〇 違い」で検索するユーザーにとって有用なコンテンツ

Googleのガイドラインでは、FAQPage構造化データの内容は**ページ上にも表示されている必要がある**とされています。つまりJSON-LDだけ書いてページに表示しないのはNGです。この2つはセットで実装する必要があります。

### チャネル3: llms.txt（AI検索向け）

llms.txtは、AIクローラー向けにサイトの概要と主要情報をMarkdown形式で提供するファイルです。

```markdown
# サイト名

> サイトの概要（1〜2文）

## 他のツールとの違い

高額SEOツール（月額$99〜$499）は
被リンク分析・大規模クロール・キーワード順位追跡など
「分析」に特化しています。
当ツールは「ページ診断→改善コード自動生成」に特化した
現場向けツールです。
```

**効果:**

- ChatGPT・Perplexity・GeminiがあなたのサイトをMarkdownで効率的に理解
- 「〇〇 とは」「〇〇 比較」とAIに質問したとき、正確に引用される確率が上がる
- `llms-full.txt`（詳細版）を併設すれば、AIがより深い情報まで参照可能

## なぜ3面展開なのか — 1つのコンテンツで3つの流入経路

ここが戦略の核心です。

FAQ自体は1回だけ作ればいい。同じ「質問と回答」のセットを、**3つの形式に変換して配置する**だけです。

| 対象                        | 形式     | 配置場所                      | 工数     |
| --------------------------- | -------- | ----------------------------- | -------- |
| Google（構造化データ）      | JSON-LD  | `<head>`内の`<script>`        | 初回のみ |
| ユーザー+Google（テキスト） | HTML     | ページ本文                    | 初回のみ |
| AI検索                      | Markdown | `/llms.txt`, `/llms-full.txt` | 初回のみ |

**3つの形式を同期して管理する**のがポイントです。FAQ内容を更新したら、3箇所すべてに反映します。

実装上のコツとしては、FAQデータを1箇所で定義し、そこからJSON-LD・HTMLコンポーネント・llms.txtの3つを生成する設計にすると、同期漏れを防げます。

## 競合ブランド名との関連構築

3面展開の効果が特に大きいのが、**競合ブランド名を含む検索クエリ**への対応です。

ユーザーは以下のような検索をします。

- 「〇〇ツール 代わり 安い」
- 「〇〇 より詳しい ツール」
- 「〇〇 無料 代替」
- 「〇〇 vs △△ 比較」

これらの検索意図に対して、FAQの中で比較情報を提供します。

```
Q: 他のSEOツールとの違いは？
A: 高額ツールは被リンク分析・順位追跡に特化しています。
   当ツールは「ページ診断→改善コード自動生成」に特化しており...
```

このFAQが3チャネルすべてに展開されることで、以下の効果が生まれます。

- **Google検索**: FAQリッチリザルトで比較検索の結果に表示
- **ページ内**: 比較検討中のユーザーの疑問に直接回答
- **AI検索**: 「〇〇の代わりになるツールは？」というAIへの質問で引用

ポイントは、**競合を貶めるのではなく、違いを客観的に説明する**こと。各ツールの強みを認めた上で、自分のツールが得意な領域を明確にします。

## 実装の手順

自分のサイトで3面展開を始めるための手順です。

### Step 1: FAQコンテンツを設計する

まず、ユーザーが実際に持つ疑問をリストアップします。

- サービスの使い方に関する質問
- 料金・プランに関する質問
- **競合との違いに関する質問**（ここが重要）
- 技術的な仕組みに関する質問

Google Search Consoleの検索クエリや、サイト内検索のログがあれば、実際にユーザーが何を知りたがっているかがわかります。

### Step 2: JSON-LD FAQPageを実装する

`<head>`内にFAQPage構造化データを追加します。

```html
<script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "FAQPage",
    "mainEntity": [
      {
        "@type": "Question",
        "name": "質問文",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "回答文"
        }
      }
    ]
  }
</script>
```

実装後は[Googleのリッチリザルトテスト](https://search.google.com/test/rich-results)で検証してください。

### Step 3: ページ上にFAQセクションを追加する

JSON-LDと同じ内容をページ本文にも表示します。アコーディオン形式が一般的です。

### Step 4: llms.txtを作成・配置する

サイトルートに`/llms.txt`を配置します。FAQの内容（特に競合比較セクション）を含め、AIクローラーが効率的にサイトを理解できるようにします。

詳細な情報が必要な場合は`/llms-full.txt`も併設します。

### Step 5: 3つの同期を維持する

FAQ内容を更新したら、3箇所すべてに反映されるようにします。手動管理でも可能ですが、FAQデータを1箇所で管理して3つの出力を生成する仕組みにするのが理想です。

## 効果の確認方法

3面展開の効果は以下の指標で確認できます。

- **Google Search Console**: FAQリッチリザルトの表示回数・クリック率の変化
- **GA4**: 競合ブランド名を含む検索クエリからの流入（UTMパラメータで追跡）
- **Google検索**: 自サイト名で検索してリッチリザルトが表示されるか確認
- **AI検索**: ChatGPTやPerplexityで「〇〇 おすすめ」「〇〇 比較」と聞いて、自サイトが引用されるか確認

特にAI検索での効果測定はまだ定量的なツールが確立されていないため、手動での確認が中心になります。これは業界全体の課題でもあります。

## 2026年以降の展望

AI検索の普及速度を考えると、3面展開は「やったほうがいい」から「やらないと取りこぼす」フェーズに移行しつつあります。

今後の注目ポイントは以下の3つです。

1. **llms.txtの標準化**: まだデファクトスタンダードが確立していない段階。早期に設置しておくことで先行者利益を得られる
2. **AI Overviewの拡大**: GoogleがAI Overviewの表示範囲を拡大する中で、構造化データの重要性はさらに増す
3. **AI検索の計測ツール**: 現状ではAI検索からの流入を正確に計測する手段が限られているが、今後ツールが整備されるはず

いずれにせよ、FAQコンテンツを3面で展開するコストは低く、リスクはほぼゼロです。「やっておいて損はない」施策です。

## まとめ

- AI検索の台頭で、SEO（Google向け）だけでなくAEO・LLMOへの対応が必要になっている
- 同じFAQコンテンツを**JSON-LD / ページHTML / llms.txt**の3形式で展開することで、3つの流入経路をカバーできる
- 特に競合ブランド名を含む比較FAQは、3チャネルすべてで効果が大きい
- FAQデータを1箇所で管理し、3つの出力を生成する設計が同期漏れを防ぐコツ
- 実装コストは低く、既存のコンテンツを再利用するだけで始められる
