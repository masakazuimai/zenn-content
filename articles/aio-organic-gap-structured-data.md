## オーガニック圏外でもAIOに引用される現象

Google検索で「seo スコア チェック 無料」と入力すると、AI Overview（AIO）の回答欄に自分が運営するSEOツールが1番目に引用されていました。

しかし、オーガニック検索結果の1ページ目に当サイトは表示されていません。上位はohotuku.jp、EmmaTools、itomakihitode.jpといった老舗ツールが占めています。

サイト公開から約3ヶ月、ドメイン歴も浅く被リンクも少ない。オーガニックで勝てないのは当然です。にもかかわらず、AIによる引用では最上位に選ばれている。このギャップは偶然なのか、それとも構造的な要因があるのか。

本記事では、実際の実装コードを交えてAIO引用の要因を分析します。

## 外部調査データ：AIO引用とオーガニック順位は別の基準

この現象は当サイト固有のものではありません。

**Ahrefsの86.3万SERP分析（2024年）** によると：

- AIO引用の **62%** がオーガニックトップ10以外のページから選ばれている
- AIOが引用するページの平均DR（Domain Rating）は **65** — 巨大サイトでなくても引用される
- AIOは「順位が高いページ」ではなく「回答に適した情報を持つページ」を優先する

**Seer Interactiveの分析** では、AIO引用の約 **55%** が各ページの上部30%から情報を抽出していることも判明しています。ページ冒頭で検索意図に直接回答する構造が、AIO引用の重要な条件です。

## 要因1：llms.txtの実装

llms.txtは、LLM（大規模言語モデル）がサイトの概要を効率的に取得するためのテキストファイルです。robots.txtのAI版と考えるとわかりやすいです。

サイトルートに設置した `llms.txt` の構成はこのようになっています。

```markdown
# サービス名

> サービスの概要を1段落で記述。ツールの目的、主な機能、
> 差別化ポイントをここにまとめる。

## サービス概要

具体的な説明。チェック項目数、対応カテゴリ数など
定量的な情報を含める。

## 主な機能

### SEO診断
- SEOスコアチェック: 項目数と採点基準
- 構造化データ診断: 対応スキーマの種類
- Core Web Vitals測定: 測定方法と基準値

### キーワード調査
- Googleサジェスト一括取得
- 検索ボリューム表示
- 競合キーワード調査

## 料金プラン

- 無料プラン: ¥0 — 機能の概要
- 有料プラン: ¥X,XXX/月 — 追加される機能

## 他のSEOツールとの違い

競合ツールとの比較、独自の価値提案を明記する。

## 詳細情報

- [llms-full.txt](https://example.com/llms-full.txt): 全機能の詳細
```

ポイントは3つです。

1. **冒頭の引用ブロック（`>`）** でサービス概要を端的に記述する
2. **定量的な情報**（項目数、カテゴリ数、料金）を含める
3. **差別化ポイント**を「他のツールとの違い」セクションで明記する

これにより、AIがサイトの概要・機能・独自性を1ファイルで把握できます。設置コストはテキストファイル1つの作成のみです。

## 要因2：JSON-LDによる構造化データ

構造化データの実装量が、AIO引用に寄与した最大の要因と考えています。

### SoftwareApplicationスキーマ

ツール型サイトでは `SoftwareApplication` を実装することで、Googleにサービスの性質を機械的に伝えられます。

```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "SEO診断ツール",
  "applicationCategory": "WebApplication",
  "operatingSystem": "Web",
  "offers": [
    {
      "@type": "Offer",
      "price": "0",
      "priceCurrency": "JPY",
      "name": "無料プラン",
      "description": "月3回のSEO診断"
    },
    {
      "@type": "Offer",
      "price": "980",
      "priceCurrency": "JPY",
      "name": "エントリープラン",
      "description": "月30回のSEO診断、検索ボリューム表示"
    }
  ],
  "featureList": [
    "45項目以上のSEO診断",
    "構造化データ検出・検証",
    "Core Web Vitals測定",
    "改善コード自動生成"
  ]
}
```

`featureList` に具体的な機能を列挙しておくと、AIが「このツールは何ができるか」を正確に抽出できます。AIOの引用文に「45項目を診断」「改善コードも自動生成」が含まれていたのは、このスキーマが情報源になっていた可能性があります。

### FAQPageスキーマ

FAQは、AIがQ&A形式で情報を抽出する際に最も利用しやすい構造化データです。

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "SEOチェックとは何ですか？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Webサイトが検索エンジンに最適化されているかを総合的に確認する作業です。タイトルタグ・メタディスクリプション・構造化データなど、検索順位に影響する要素を分析します。"
      }
    },
    {
      "@type": "Question",
      "name": "無料で使えますか？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "はい、登録不要で3回まで無料で利用可能です。アカウント登録で毎月リセットされます。"
      }
    }
  ]
}
```

FAQPageの設計で重要なのは、**ユーザーが実際に検索しそうな質問文**をそのまま `name` に使うことです。「Q1」のような記号ではなく、自然言語の質問文にします。

## 要因3：ページ冒頭の直接回答

Seer Interactiveの調査で「AIO引用の55%がページ上部30%から抽出される」と報告されています。

つまり、ページの冒頭で「このサービスは何をするか」を具体的に書くことが重要です。

```html
<meta
  name="description"
  content="URLを入力するだけで、構造化データ・メタタグ・技術SEOなど
  45項目以上を無料で瞬時に診断。改善コード自動生成付き。"
/>
```

ランディングページのh1直下でも同様に、具体的な数値を含む1文で要約します。

```
URLを入力するだけで、Webサイトの SEO状況を
45項目以上で総合診断します。
```

AIOの引用文を見ると、「URL入力で45項目を診断。改善コードも自動生成。」と書かれていました。ページ冒頭の記述がほぼそのまま引用されています。

## 要因4：差別化ポイントの明記

AIOは複数ツールを比較して回答を生成します。このとき「他と何が違うか」が明記されているサイトは引用されやすくなります。

当サイトでは「改善コードも自動生成」という機能をページ上で繰り返し言及していました。AIOはこの差別化ポイントをそのまま引用文に含めています。

llms.txtでも構造化データでもメタディスクリプションでも、**同じUSPを一貫して記述する**ことで、AIが「このサイトの独自価値」を確信を持って抽出できる状態になります。

## 「GEO」は良いSEOそのもの

ここまでの要因を振り返ると、AIO引用のために特別なことは何もしていません。

- 構造化データの実装 → SEOの基本
- ページ冒頭での直接回答 → コンテンツ設計の基本
- 差別化ポイントの明記 → LPの基本
- llms.txt → robots.txtの延長

GEO（Generative Engine Optimization）という言葉が注目されていますが、その正体は「良いSEOの徹底」に他なりません。AIが引用するページと、Googleが評価するページの特徴は、構造化・明確さ・情報の抽出しやすさという点で一致しています。

特別な「GEO対策」を考えるよりも、構造化データの実装、直接的な回答構造、一貫したUSP記述といった基本を徹底するほうが、結果的にAIO引用にもつながります。

## まとめ

| 施策 | コスト | AIO引用への寄与 |
|------|--------|----------------|
| llms.txt設置 | テキストファイル1つ | AIがサイト概要を直接取得できる |
| JSON-LD（SoftwareApplication等） | 実装工数1-2時間 | 機能・料金を機械可読にする |
| ページ冒頭の直接回答 | コピー修正のみ | AIO引用の55%がここから抽出 |
| USPの一貫記述 | 設計方針の問題 | AIが差別化ポイントを確信を持って引用 |

オーガニック順位が低くても、サイトの情報がAIにとって抽出しやすい構造になっていれば、AIO引用という形で先に成果が出ることがあります。ドメイン歴の浅い新規サイトほど、この戦略は有効です。
